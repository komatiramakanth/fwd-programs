🌐 CORS — Cross-Origin Resource Sharing Demo

📖 What is CORS?

CORS is a browser security policy that blocks requests made from one origin to a different origin unless the server explicitly allows it.

🎯 API server origin: http://localhost:3000

⚠️ These are DIFFERENT origins → CORS rules apply!

❌ Demo 1 — Request WITHOUT CORS Header (Blocked)

This button fetches from http://localhost:3000/no-cors which does NOT send an Access-Control-Allow-Origin header. The browser will BLOCK the response.

✅ Demo 2 — Request WITH CORS Header (Allowed)

This button fetches from http://localhost:3000/with-cors which sends Access-Control-Allow-Origin: *. The browser will ALLOW the response.

📋 Common CORS Headers Reference

Header Example Value What it means
Access-Control-Allow-Origin * or https://example.com Which origins can read the response
Access-Control-Allow-Methods GET, POST, PUT Which HTTP methods are allowed
Access-Control-Allow-Headers Content-Type, Authorization Which request headers are allowed
Access-Control-Allow-Credentials true Allow cookies to be sent cross-origin
Access-Control-Max-Age 86400 How long to cache preflight response (seconds)
